# 刷题网站 V2 升级计划

## 需求分析
1. **已刷/未刷标记** — 首页和练习页面显示哪些题已做过
2. **进度记忆（防清零）** — 答题记录持久化，不会丢失
3. **跨设备同步** — 不同设备刷的题目都能记录

## 实现方案

### 1. 首页进度可视化
- 在首页添加进度卡片：已做/未做比例、正确率
- 四个模式卡片上显示已做/总数

### 2. 题目列表/练习页面已刷标记
- 答题卡（模拟考试）显示每题状态：已做✅/未做⬜/做错❌
- 顺序练习中显示"已做过"标记

### 3. 持久化增强
- IndexedDB 存储（比 localStorage 更可靠，容量更大）
- 答题记录包含：题目ID、用户答案、是否正确、答题时间
- 所有数据自动保存，不会丢失

### 4. 跨设备同步
- JSON 文件导出/导入功能
- 用户可设置同步码（4-6位），导出时包含同步码
- 在其他设备导入时，根据同步码合并记录
- 添加云端同步（使用 supabase 免费版）

## 执行步骤
1. 检查现有代码结构
2. 修改数据存储层（添加 IndexedDB）
3. 修改首页（添加进度显示）
4. 修改练习页面（添加已刷标记）
5. 添加导出/导入同步功能
6. 构建部署
